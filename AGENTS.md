# Repository Guidelines

## Project Structure & Module Organization

This repository is a Node ESM economic simulation workbench with a React/Vite UI.

- `src/index.js` contains the simulation engine, world state, turn advancement, metrics, and invariant checks.
- `src/agents.js` defines prompt construction and OpenAI-compatible/LongCat agent adapters.
- `src/experiments.js` and `src/storage.js` provide comparison helpers and saved-run persistence.
- `src/ui/` contains the React app: `App.jsx`, `App.css`, and `main.jsx`.
- `server.mjs` serves the API and Vite middleware for local development.
- `test/` contains `node:test` suites. `scripts/` contains smoke tests and LongCat runner scripts.
- `dist/` is generated by `npm run build`; do not hand-edit built assets.

## Build, Test, and Development Commands

- `npm run dev` or `npm start`: start the local API + Vite app at `http://localhost:5173`.
- `npm test`: run all Node test suites.
- `npm run build`: build the production client into `dist/`.
- `npm run qa:http`: smoke-test the HTTP API.
- `npm run qa:browser`: run the Playwright browser smoke test.
- `npm run simulate:llm`: run the LongCat-backed simulation script.

## Coding Style & Naming Conventions

Use modern JavaScript modules (`import`/`export`) and two-space indentation. Prefer named exports for shared engine helpers. Keep simulation state fields in snake_case when they represent contract payloads (`global_trust`, `proposal_id`) and use camelCase for JavaScript variables/functions (`globalTrust`, `createSimulationSession`). Keep UI component names in PascalCase.

## Testing Guidelines

Tests use the built-in `node:test` runner with `node:assert/strict`. Add focused tests under `test/*.test.js`, with behavior-oriented names such as `zero trust overrides an accepting agent in async simulation`. For UI or integration changes, update `scripts/smoke-browser.mjs` or `scripts/smoke-http.mjs` when the expected browser/API behavior changes. Run `npm test` before broader QA.

## Commit & Pull Request Guidelines

This checkout has no Git history available, so use clear imperative commits such as `fix trust gate for agent decisions` or Conventional Commit style (`fix: gate zero-trust decisions`). PRs should include a short problem statement, changed behavior, verification commands run, and screenshots for visible UI changes.

## Security & Configuration Tips

Keep secrets in `.env`; never commit API keys. Use `.env.example` for required variable names. LongCat configuration is read from `LONGCAT_API_KEY`, `LONGCAT_BASE_URL`, and `LONGCAT_MODEL`.
